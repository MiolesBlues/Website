What we measure

The dataset includes performance metrics (accuracy, success rate, time), resource usage (CPU, memory), cost per task, plus governance indicators (privacy, bias detection, data quality).

  • accuracy_score — correctness (0–1)
  • success_rate — success probability (0–1)
  • execution_time_seconds — speed/latency
  • cpu_usage_percent / memory_usage_mb — resources
  • cost_per_task_cents — cost at scale

Limitations & Ethics

JavaScript was not taught in class during this module. For implementation support, our team used GitHub Copilot inside Visual Studio Code, mainly to help with syntax, boilerplate structure, and small code suggestions.

However, all logical decision-making, data analysis, and interpretation of results were completed by the team members. We used Copilot as a tool for productivity, not as a replacement for understanding or academic work.

Conclusion

The results highlight practical trade-offs: higher complexity generally increases time and resource use, and accuracy can drop as tasks become harder.
